# Pali Hangul Keyboard - 한글 키 매핑 설계

## 설계 철학

1. **직관성**: 한국인이 바로 이해할 수 있어야 함
2. **속도**: 타이핑 속도를 방해하지 않아야 함
3. **확장성**: 나중에 다른 특수 문자 추가 가능해야 함

---

## 최종 매핑 방안 (안)

### Option 1: Option Key 조합 (추천) ⭐

**장점**:
- 배우기 쉬움
- 다른 IME (일본어, 중국어)와 일관성
- 타이핑 속도 빠름

**매핑표**:

| 입력 | 출력 | 유니코드 | 설명 |
|------|------|----------|------|
| `Option + a` | ā | U+0101 | 장음 a |
| `Option + i` | ī | U+012B | 장음 i |
| `Option + u` | ū | U+016B | 장음 u |
| `Option + m` | ṃ | U+1E43 | 니깔라 |
| `Option + n` | ṅ | U+1E45 | 점 위 n |
| `Option + Shift + n` | ñ | U+00F1 | 물결 n |
| `Option + d` | ḍ | U+1E0D | 점 아래 d |
| `Option + l` | ḷ | U+1E37 | 점 아래 l |
| `Option + t` | ṭ | U+1E6D | 점 아래 t |
| `Option + Shift + t` | ṇ | U+1E47 | 점 아래 n |

---

### Option 2: 한글 받침 활용

**장점**:
- 한글 자판만 사용
- Option 키 불필요

**단점**:
- 학습 곡선 높음
- 받침 충돌 가능성

**매핑표 (예시)**:

| 입력 | 출력 | 로직 |
|------|------|------|
| `ㅏ + ㄱ` | ā | 모음 + ㄱ → 장음 |
| `ㅣ + ㄱ` | ī | 모음 + ㄱ → 장음 |
| `ㅜ + ㄱ` | ū | 모음 + ㄱ → 장음 |
| `ㅁ + ㅂ` | ṃ | 자음 + ㅂ → 점 아래 |
| `ㄴ + ㄱ` | ṅ | 자음 + ㄱ → 점 위 |
| `ㄴ + ㅁ` | ñ | 자음 + ㅁ → 물결 |

---

### Option 3: 쌍자음/쌍모음

**장점**:
- 직관적 (더블 타이핑)
- 받침 충돌 없음

**단점**:
- 타이핑 속도 느림
- 실수 가능성 높음

**매핑표 (예시)**:

| 입력 | 출력 |
|------|------|
| `ㅏㅏ` | ā |
| `ㅣㅣ` | ī |
| `ㅜㅜ` | ū |
| `ㅁㅁ` | ṃ |
| `ㄴㄴ` | ṅ |

---

## 최종 권장 사항

**Option 1 (Option Key 조합)**을 추천합니다!

**이유**:
1. 국제 표준에 가까움 (다른 언어 IME와 일관성)
2. 빠르고 정확함
3. 확장 가능성 높음

---

## 구현 계획

### 1단계: 기본 문자 (10개)
- ā, ī, ū (장음 3개)
- ṃ, ṅ, ñ (비음 3개)
- ḍ, ḷ, ṭ, ṇ (경구개음 4개)

### 2단계: 확장 문자 (미래)
- 대문자 변형 (Ā, Ī, Ū 등)
- 기타 조합 문자
